# 选题路线图：跨形态控制、快速适应与上下文建模

这份文档面向一个具体目标：在 `RMA` 和 `MetaMorph` 的基础上，找出**2024~至今仍然相对空缺、且更容易投机器人/AI 顶会**的研究切口。

我先给一个结论版判断：

- 纯粹的“多形态统一控制器”已经被较多近年工作覆盖。
- 纯粹的“在线适应/快速恢复”也已经有不少工作。
- 真正还比较值得做的，是把两者统一成一个更清晰的问题：**跨形态前提下的时变上下文推断与测试时适应**。

下面把可行路线整理成三条主线，再加两个横切方向：**上下文路由** 和 **可迁移上下文表示**。

---

## 1. 近年相关工作给出的边界

以下工作说明“老版本路线”哪些地方已经被覆盖：

- [One Policy to Run Them All](https://arxiv.org/abs/2409.06366) / [OpenReview 版本](https://openreview.net/pdf?id=PbQOZntuXO)
  - 已经覆盖了“多 embodiment locomotion + 统一策略 + 泛化到新形态”。
- [Universal Morphology Control via Contextual Modulation](https://openreview.net/forum?id=PzlO3SSk3A)
  - 已经覆盖了“形态 context 显式调制策略”。
- [Distilling Morphology-Conditioned Hypernetworks for Efficient Universal Morphology Control](https://openreview.net/forum?id=WjvEvYTy3w)
  - 已经覆盖了“形态条件化控制器的压缩/蒸馏”。
- [OCCAM: Online Continuous Controller Adaptation with Meta-Learned Models](https://openreview.net/forum?id=VBKgW9jzeZ)
  - 已经覆盖了“在线 controller adaptation”。
- [Modular Recurrence in Contextual MDPs for Universal Morphology Control](https://openreview.net/forum?id=0fn0ii1njp)
  - 已经覆盖了“上下文可通过交互推断，并用于通用 morphology control”。
- [LocoFormer: Generalist Locomotion via Long-context Adaptation](https://openreview.net/forum?id=VqmAvBkFhw)
  - 已经覆盖了“长上下文记忆 + 跨 episode 适应 + generalist locomotion”。
- [Scaling Cross-Embodied Learning: One Policy for Manipulation, Navigation, Locomotion and Aviation](https://arxiv.org/abs/2408.11812)
  - 已经说明“跨 embodiment 的大一统 policy”这个方向越来越像 foundation model 叙事。

因此，**你现在的切入点不能只是“把 RMA latent 接到 MetaMorph 上”**。  
更好的方式是把问题重新定义为：

> 跨形态控制中，哪些上下文应显式建模，哪些上下文应通过历史在线推断，二者如何解耦，以及如何在部署时快速适应。

## 2. 三条主路线

### 路线 A：分层上下文建模 for Cross-Embodiment Adaptation

这是我最推荐的主线。

#### 核心问题

形态信息和自身参数变化、环境扰动到底应该如何建模？

一个更稳的答案不是“全塞进一个 latent”，而是分成两层：

- `c_static`
  - morphology、拓扑、关节类型、模块组合、硬件静态属性
- `c_dynamic(t)`
  - 质量变化、摩擦、motor strength、payload、terrain、接触状态、损伤等时变上下文

然后：

- `z_m = E_static(c_static)`
- `z_d(t) = E_dyn(history)`
- policy = `f(token, z_m, z_d(t))`

#### 为什么它还值得做

近年的工作已经做了“统一 morphology”或“在线适应”，但**把静态形态和时变动力学明确拆开，并在跨形态控制里统一起来**，仍然比较有研究价值。

#### 适合的贡献点

- 证明“静态形态”和“时变物理上下文”混在一起会伤泛化。
- 证明分层上下文能提升 unseen morphology 和 unseen dynamics 下的稳定性。
- 给出可解释的 latent 分工。

#### 主要风险

- 如果 `c_static` 和 `c_dynamic` 划分太武断，审稿人会觉得是工程切分，不是方法。
- 需要做足消融，证明分层不是偶然收益。

#### 适合的投稿口味

- Robotics learning 顶会很合适。
- 如果实验足够系统，也能往 broader AI 方向靠。

---

### 路线 B：Test-Time Adaptation for Universal Modular Controllers

这条路线更像 `RMA` 的精神，但你要把它放到跨形态场景里。

#### 核心问题

一个 universal controller 能否在部署时，仅凭短历史快速恢复到合适的控制模式？

这里的重点不再只是“控制不同 robot”，而是：

- 新 morphology
- 新 dynamics
- 新 terrain
- 新损伤

全部在 test time 处理。

#### 建议叙事

把 `MetaMorph` 当作 universal policy backbone，把 `RMA` 风格的 history encoder 当作 adaptation module。

#### 为什么这条线还空

已有工作大多要么偏统一 morphology，要么偏 controller adaptation。  
你可以把贡献聚焦成：

- **跨形态前提下的 test-time rapid adaptation**
- **没有梯度更新或少梯度更新的 adaptation**
- **恢复速度作为核心指标**

#### 适合的贡献点

- 进入新环境后的恢复曲线
- 进入新 morphology 后的 few-shot / zero-shot recovery
- history encoder 学到的 latent 对不同机器人是否共享语义

#### 主要风险

- 容易被质疑“只是更大的 context window”。
- 需要严格控制模型规模与 history 长度，避免被说成堆参数。

#### 适合的投稿口味

- 机器人学习顶会非常合适。
- 如果强调 test-time inference，也可以向更广义 AI 叙事靠。

---

### 路线 C：Long-Context Cross-Embodiment Locomotion

这是更偏近期趋势的一条线，和 `LocoFormer` 的方向最接近。

#### 核心问题

当前的 universal locomotion 是否过于“短视”？

如果把 context 扩到跨 episode、跨扰动恢复、跨 morphology history，能否让策略表现出 emergent adaptation？

#### 为什么有潜力

近年的一个明显趋势是：

- 短历史不够
- 长历史和 episode boundary 外的记忆开始重要
- 适应行为可能不是一步完成，而是跨多个 episode 逐步形成

#### 可行的创新点

- 长 context memory
- cross-episode adaptation
- morphology + dynamics + terrain 的联合记忆

#### 主要风险

- 训练更难
- 工程复杂
- 很容易变成“一个更大模型”

#### 适合的投稿口味

- 比较适合想往 broad AI / foundation model 叙事靠近的路线。
- 机器人方向也能投，但需要很强的 benchmark 和分析。

## 3. 两个横切方向

### 3.1 上下文路由

这个方向比“上下文拼接”更有方法感。

#### 含义

不是把 context 当作普通输入，而是让 context 决定：

- 哪些模块该激活
- 哪些 limb 共享参数
- 哪些注意力边应该更强
- 哪些 residual/gate 应该打开

#### 为什么值得做

因为近年很多 morphology control 方法已经证明，**单纯把 morphology context 作为 bias 加进去不一定够**。  
上下文路由比纯拼接更像一个“结构性条件控制”机制。

#### 可能的实现方式

- context-conditioned hypernetwork
- context-conditioned attention mask
- context-conditioned mixture-of-experts
- per-limb adaptive gate

#### 更像论文的点

- 不只是“有 context”
- 而是“context 如何改变计算路径”

#### 风险

- 结构复杂后，创新点会被审稿人要求讲得很清楚。
- 如果没有强消融，会显得像把多个机制混在一起。

---

### 3.2 可迁移的上下文表示

这个方向适合做成“表示学习”层面的贡献。

#### 含义

你不是只关心这个 latent 能不能让当前机器人走得更好，而是想问：

- 同一个 latent 在不同 morphology 间是否语义一致？
- 同一个 context representation 能否迁移到新 robot？
- 它是否能作为一个通用的 adaptation interface？

#### 为什么这条线有价值

很多方法都用 context latent，但很少系统地回答：

- latent 是否可解释
- latent 是否可复用
- latent 是否跨 embodiment 对齐

#### 可做的实验

- 同一 latent 下，不同 morphology 的行为相似性
- latent 插值是否对应连续的控制变化
- 新 robot 上 latent probing
- 跨任务 / 跨 terrain 的复用

#### 更像论文的点

- 从“控制性能”提升到“context 表示的可迁移性”

#### 风险

- 如果没有好的 probing 或分析，容易显得偏辅助性质，不够主线化。

---

## 4. 形态和自身参数要不要分开

我的建议是：

- **语义上要分开**
- **实现上可以先统一，再做解耦**

### 为什么语义上最好分开

因为两类信息的时间尺度不同：

- 形态是慢变量，通常 episode 级甚至更长时间尺度不变。
- 自身参数和环境扰动很多是快变量，可能 step 级变化。

把它们混成一个大 context，短期可能能跑，但会让模型难以学到稳定分工。

### 为什么实现上可以先统一

因为工程上你最先需要的是一个可训练、可验证的系统。

所以可以先：

- 用两个 encoder
- 输出两个 slot
- 在 transformer 内融合

这样你既保留了统一接口，又保留了解耦空间。

### 我更推荐的折中

第一版就做成：

- `z_m`：morphology slot
- `z_a(t)`：adaptation slot

然后再做消融：

- 只用 `z_m`
- 只用 `z_a(t)`
- `z_m + z_a(t)`

这会比“一个大 latent”更有论文感，也更容易回答审稿人的问题。

---

## 5. 我建议你优先尝试的组合

如果目标是更容易投机器人顶会，我建议主线是：

**路线 A + 路线 B**

也就是：

- 分层上下文建模
- test-time rapid adaptation

这是最稳的，因为它同时满足：

- 和 `RMA` 近
- 和模块化机器人近
- 和近期文献有差异
- 还能讲清楚为什么要分层

### 不建议一开始就主打的方向

- 纯上下文路由，风险高，容易复杂化
- 纯可迁移表示，容易变成分析型附属贡献
- 纯长上下文，可能会被说成“更大的模型”

这些方向可以作为增强点，但不建议作为第一主线。

---

## 6. 如果你要做成顶会稿子，最关键的三个问题

1. 你的上下文分层依据是什么？
   - 时间尺度
   - 物理语义
   - 可观测性

2. 你的 adaptation 到底在适应什么？
   - morphology
   - dynamics
   - terrain
   - damage

3. 你的 latent 是否真的可迁移？
   - 跨机器人
   - 跨环境
   - 跨 episode

只要这三个问题答得清楚，路线一就不再只是“老项目融合”，而是一个能往新论文走的研究方向。

---

## 7. 建议的下一步

如果继续推进，我建议下一版文档或者实验计划直接围绕下面三个具体方案展开：

1. `z_m + z_a(t)` 的分层上下文版
2. context routing 版
3. transferable context representation 版

这三者可以分别对应：

- 主论文主线
- 强化方法感
- 强化分析和解释性
