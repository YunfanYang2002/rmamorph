# 运行模式命令与命名约定

这份文档专门记录当前项目里常用的运行模式命令。以后如果你要切换 `baseline / teacher / student / hybrid`，优先看这份文档。

## 1. 统一规则

当前有两个最重要的开关：

- `MODEL.CONTEXT_MODE`
- `MODEL.WRAPPERS`

其中：

- `MODEL.CONTEXT_MODE = none` 表示旧版基线
- `MODEL.CONTEXT_MODE = teacher` 表示带显式 context，但不走 history student
- `MODEL.CONTEXT_MODE = student` 表示走 history-based student 路线
- `MODEL.CONTEXT_MODE = hybrid` 表示 teacher + student 都保留

如果要启用 history，还需要：

- `MODEL.WRAPPERS = ['HistoryContextWrapper']`

## 2. 常用命令

下面的命令默认都以 PowerShell 为例。

### 2.1 旧版基线

```powershell
$env:CUDA_VISIBLE_DEVICES='0'
python .\tools\train_ppo.py --cfg .\configs\ft.yaml `
  MODEL.CONTEXT_MODE none `
  MODEL.WRAPPERS "[]"
```

适合：

- 和新结构做 baseline 对照
- 看没有 context 和 history 时的表现

### 2.2 teacher 模式

```powershell
$env:CUDA_VISIBLE_DEVICES='0'
python .\tools\train_ppo.py --cfg .\configs\ft.yaml `
  MODEL.CONTEXT_MODE teacher `
  MODEL.WRAPPERS "[]"
```

适合：

- 测试 `static_context` / `adaptive_context` 是否有帮助
- 不引入 history，先看纯 context 版本的效果

### 2.3 student 模式

```powershell
$env:CUDA_VISIBLE_DEVICES='0'
python .\tools\train_ppo.py --cfg .\configs\ft.yaml `
  MODEL.CONTEXT_MODE student `
  MODEL.WRAPPERS "['HistoryContextWrapper']"
```

适合：

- 验证 history-based adaptation 是否有效
- 看短历史能不能帮助策略恢复

### 2.4 hybrid 模式

```powershell
$env:CUDA_VISIBLE_DEVICES='0'
python .\tools\train_ppo.py --cfg .\configs\ft.yaml `
  MODEL.CONTEXT_MODE hybrid `
  MODEL.WRAPPERS "['HistoryContextWrapper']"
```

适合：

- 同时保留 teacher 和 student 通路
- 做更完整的对照实验

## 3. TensorBoard 命名规则

现在 TensorBoard 会自动按模式分层，路径会长成这样：

```text
output/tensorboard/<mode_tag>/run_lr..._gamma..._enc..._ac..._<mode_name>_<timestamp>
```

例如：

```text
output/tensorboard/student_history/run_lr..._student_history_20260528_113000
```

其中 `mode_tag` 是由代码自动生成的：

- `baseline_nohistory`
- `teacher_history`
- `student_history`
- `hybrid_history`

## 4. 如何理解这些目录

你以后看 TensorBoard 时，先看最外层目录：

- `baseline_nohistory`：旧版/对照组
- `teacher_history`：显式 context 版本
- `student_history`：history adaptation 版本
- `hybrid_history`：teacher + student 版本

这样做的好处是：

- 目录结构直接告诉你当前实验在做什么
- 后面不会把不同模式的曲线混在一个文件夹里
- 新加功能时也更容易同步管理

## 5. 推荐的实验顺序

如果你要做一轮完整验证，建议按这个顺序跑：

1. `baseline`
2. `teacher`
3. `student`
4. `hybrid`

这样最容易看出：

- context 是否有帮助
- history 是否有帮助
- 两者同时启用是否更稳
